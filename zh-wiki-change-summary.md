# 中文 Wiki 功能改动总结

日期：2026-06-07

## 目标

在保持 GitNexus 原有 wiki 生成功能不变的前提下，增加可选的简体中文 wiki 生成能力。

启用方式：

```bash
gitnexus wiki --lang zh-CN --force
gitnexus wiki --lang 中文 --force
gitnexus wiki --lang chinese --force
```

默认不传 `--lang` 时，原有英文 wiki 行为保持不变。

## 功能行为

中文模式会本地化这些可见内容：

- 页面标题和 H1
- 模块显示名
- 文档正文
- 小节标题
- Mermaid 图表标签
- HTML viewer 中的基础 UI 文案

中文模式会保持这些源代码相关内容原样：

- 函数名、类名、变量名、真实 API 名称
- 文件路径、import path、包名
- CLI 命令、配置 key
- 公式、复杂度表达式、数学符号
- Markdown 链接目标和页面 slug

## 主要实现

### `gitnexus/src/core/wiki/generator.ts`

- 新增中文语言归一逻辑，将 `chinese`、`zh`、`zh-CN`、`中文`、`简体中文` 统一为 `zh-CN`。
- 修改 `buildSystemPrompt()`，中文模式下要求自然语言输出中文，同时明确保留代码标识符、路径、命令、公式和链接目标。
- 新增模块显示名本地化流程：
  - 分组阶段仍保持原逻辑，避免破坏 JSON 结构和文件路径稳定性。
  - 分组完成后，在中文模式下额外调用一次 LLM，将模块显示名翻译成简体中文。
  - `slug` 保持原 ASCII 值，便于链接、HTML hash 和增量更新稳定。
- 修改页面内容组装逻辑：
  - LLM 返回首行 H1 时使用该标题。
  - LLM 未返回 H1 时使用模块显示名作为 fallback。
  - 避免重复插入 H1。
- 增量更新查找逻辑改为支持按 `name` 或 `slug` 查找模块，兼容中文显示名。

### `gitnexus/src/core/wiki/prompts.ts`

- 给父模块页 prompt 增加真实子页面链接列表。
- 给 overview prompt 增加真实模块链接列表。
- 目标是减少中文标题下 LLM 错写 `.md` 链接目标的概率。

### `gitnexus/src/core/wiki/html-viewer.ts`

- 根据 `meta.lang === "zh-CN"` 切换 HTML viewer 文案。
- 中文模式下输出 `<html lang="zh-CN">`。
- 本地化文案包括：
  - `Overview` -> `概览`
  - `Modules` -> `模块`
  - `Generated by GitNexus` -> `由 GitNexus 生成`
  - `Loading...` -> `加载中...`
  - `Page not found` -> `页面不存在`

### CLI help 和 README

- `--lang` help 示例增加 `zh-CN` 和 `中文`。
- 根 README 和包内 README 增加中文 wiki 使用示例。
